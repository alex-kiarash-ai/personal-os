# /venture-sync - Mirror venture docs into vault, generate briefs, scaffold project pages

Pulls high-value markdown from each external venture repo into `vault/ventures/{name}/docs/`, generates a synthesis `brief.md` per venture, and scaffolds `projects/{name}.md` pages from auto-discovered sub-initiatives. Read-only mirror. Source of truth stays in the repo.

## When this runs

- **Manually** when the user types `/venture-sync`.
- **On-demand only for now.** A weekly Monday schedule is planned but NOT yet registered (no `PersonalOS-venture-sync` job, no `scheduler/schedule.md` entry). Add both when the ventures go live. Project #19, currently DORMANT.
- **Suggested by `/lint`** if synced docs' git_sha is more than 14 days behind the source.

## Modes

- `/venture-sync` - Sync everything in the config.
- `/venture-sync <venture>` - Sync just one venture.
- `/venture-sync --discover-only` - Scan source files, propose project list, do NOT write briefs or docs. For first-run review.
- `/venture-sync --briefs-only` - Regenerate `brief.md` for every venture without re-mirroring docs.

## Step 0: Read config

Read `vault/ventures/_sync-config.md`. Parse the YAML block. For each venture entry: `code_path`, `sync_files`, `project_source`.

If `code_path` doesn't exist on disk, log a warning and skip that venture.

## Step 1: Mirror whitelisted docs

For each venture, for each filename in `sync_files`:

1. Resolve absolute source path: `{code_path}/{filename}` (expand `~`).
2. Skip if the file matches the auto-skip list (`CLAUDE.md`, `AGENTS.md`, `AGENT.md`, `CODEX.md`, `GEMINI.md`, `README.md`, oversized `CHANGELOG.md`).
3. Compute short git SHA at `code_path` HEAD: `git -C {code_path} rev-parse --short HEAD`.
4. Read source content.
5. Write to `vault/ventures/{venture}/docs/{filename}` with frontmatter prepended:
   ```yaml
   ---
   source: <absolute path>
   synced: YYYY-MM-DD
   git_sha: <short>
   read_only: true
   ---
   ```
6. If a synced file from a previous run is no longer in the whitelist, delete it from the docs/ folder.

After all venture files are mirrored, write `vault/ventures/{venture}/docs/_sync.md` (manifest):
```markdown
# Sync Manifest - {venture}

Last sync: {YYYY-MM-DD HH:MM}
Source repo: {code_path}
Git SHA at sync: {short}

| File | Source | Synced | SHA |
|------|--------|--------|-----|
| TODOS.md | ~/Desktop/AlphaStar/TODOS.md | 2026-04-26 | a3f2c1 |
```

## Step 2: Discover projects (sub-initiatives)

For each venture with a `project_source` set:
1. Read the source file (e.g. `~/Desktop/brandmodal/plan.md`).
2. Extract every `## ` heading (H2). These are project candidates.
3. Slugify each heading (lowercase, dashes, no punctuation).
4. Check `vault/ventures/{venture}/projects/{slug}.md` - if it exists, leave it (preserve user edits). If not, scaffold:

```yaml
---
tags: [project, {venture}]
venture: {venture}
status: in-progress
owner: kukesh
priority: P2
created: YYYY-MM-DD
updated: YYYY-MM-DD
source_heading: "{original heading text}"
source_file: {project_source}
---

# {Heading text}

> Auto-scaffolded from {project_source}. Edit freely.

## Current state
TBD - fill in.

## Open work
{bulleted list extracted from the body of this H2 section in source_file}

## Decisions log
- {created date}: scaffolded by /venture-sync

## Recent activity
- (regenerated each sync - see brief.md)

## Related
[[ventures/{venture}]]
```

If a previous project page exists but its source heading is gone (someone removed it from plan.md), don't delete the page - just append a `## Sync Note` flagging "source heading removed YYYY-MM-DD; mark archived if obsolete".

## Step 3: Generate brief.md per venture

Write `vault/ventures/{venture}/brief.md` with these sections (regenerated every sync - overwrite freely, this is the only auto-generated file):

```yaml
---
tags: [brief, {venture}]
venture: {venture}
generated: YYYY-MM-DD HH:MM
auto: true
---

# {Venture} - Brief

> Auto-generated by /venture-sync. Don't edit; edits will be lost on next sync. Edit status.md or the source files instead.

## Snapshot
{First paragraph from status.md}

## Current stage
{Extract "Current Stage" or "What It Is" section from status.md, or first 3 lines}

## Active projects
{For each vault/ventures/{venture}/projects/*.md with status=in-progress:}
- [[projects/{slug}]] - {short description from page} | owner: {owner} | priority: {priority}

## Recent commits (last 7 days)
{git -C {code_path} log --since='7 days ago' --oneline | head -10}
{If empty: "No commits in last 7 days."}

## Open TODOs (parsed from synced docs)
{Scan vault/ventures/{venture}/docs/*.md for lines matching `- [ ]`. Group by source file. Show top 15.}

## Recent meetings tagged {venture}
{Scan vault/meetings/*.md for frontmatter `ventures: [...{venture}...]` updated in last 30 days.}
{If none: "No recent meetings."}

## Synced docs
{Link to each file in vault/ventures/{venture}/docs/}
- [[docs/TASKS]] - synced {date}, sha {short}

## Related
[[ventures/{venture}/status]] · [[business/...]] · [[people/...]]
```

## Step 4: Personal Ops System automations get briefs too

For each `vault/projects/{name}/status.md` (the 10 automations), generate `vault/projects/{name}/brief.md`:

```yaml
---
tags: [brief, automation]
project: {name}
generated: YYYY-MM-DD HH:MM
auto: true
---

# {Name} - Brief

> Auto-generated by /venture-sync.

## What it does
{First paragraph from status.md}

## Notion DB
- ID: {db_id from status.md frontmatter}
- Last run: {last_run from status.md or "never"}

## Recent runs
{Scan vault/projects/{name}/history/ or runs/ or standups/ for files in last 7 days. List most recent 5.}

## Connections
{From status.md or work/{nn}-{name}/CLAUDE.md "Connections" section}

## Related
[[projects/{name}/status]]
```

## Step 5: Update index and log

- Append every newly created page to `vault/index.md` under the appropriate section (Ventures, Projects).
- For pages that already exist, leave the index entry alone.
- Append to `vault/log.md`: `## [YYYY-MM-DD HH:MM] /venture-sync | {N synced, M projects scaffolded, K briefs}`.

## Step 6: Report

In soul.md voice, summarize what changed:
- Ventures synced (with file counts).
- New project pages scaffolded (paths).
- Files removed from previous syncs.
- Briefs regenerated.
- Anything broken (missing repos, git failures, oversized files skipped).

## What /venture-sync does NOT do

- Never edits the source repos. Read-only on `~/Desktop/{venture}/`.
- Never modifies user-authored fields in project pages (status, decisions, current state). Only the auto-managed sections in brief.md and the scaffolded skeleton on first creation.
- Never touches `vault/ventures/{venture}/status.md` - that's user-authored.
- Never deletes a project page even if its source heading is gone (just flags it).
- Does not run `/ingest` on the synced docs (they live in their own zone, not vault/sources/).
