5. English only.
6. Deterministic dash scan (— and –, the actual em/en dash codepoints). Any hit = rewrite the sentence before showing.
7. Apply the never-share list. Check every name and number against it. The material plan already excludes run times, dates, and technical codes; do not add them back.
8. End the draft with the suggested real screenshot (from the plan; from screenshots/ or to be captured).
9. Save to episodes/episode-NN-slug.md, create/update the Content Library row (Title "Building Alex ENN: ...", Platform LinkedIn, Type post, Source "Building Alex series", Status Draft, Publish Date = slot, FULL text in page body).
10. STOP for Shaheen's review. Harvest his edits into soul.md before marking Approved. Mark the episode's row in the material plan as drafted.
11. Post-run: plan.md status column, vault/projects/linkedin-series/status.md, vault/log.md.
