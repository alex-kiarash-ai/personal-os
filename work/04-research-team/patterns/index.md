# Pattern Library - Research Team

Reusable team architectures from past research runs. Check here BEFORE designing a new team. Format spec in work/04-research-team/CLAUDE.md.

| Pattern | Class | Times used | One-liner |
|---------|-------|------------|-----------|
| [science-explainer](science-explainer.md) | other | 1 | "How/why does X work" question that splits into 2 independent science sub-questions; two parallel lanes + synthesizer that separates fact from folklore. |
| [other-flight-search](other-flight-search.md) | other | 1 | "Cheapest flight" live lookup; no team. Kiwi aggregator with flex-date tiling, parse the saved JSON locally, nonstop = 1 layover entry, filter by stay nights, rank by price. |
| [decision-brief-concept-validation](decision-brief-concept-validation.md) | decision-brief | 3 | "Validate this idea/spec, up/downside, worth building?" Verify only the load-bearing claims the author asserted but didn't check - 3 lanes: cost/viability, prior-art/sellability, core-mechanism (does the "it learns/compounds" claim survive ML/stats reality). Correct your own prior in synthesis. |
| [technical-evaluation-internal-audit-stack-scan](technical-evaluation-internal-audit-stack-scan.md) | technical-evaluation | 4 | "Build X on our own system: what of ours feeds it + what infra signals exist + best external tooling?" Two Explore audit lanes (projects, infra) + one web Stack Scout given an explicit hypothesis to falsify. Synthesizer ranks v1 vs bench and resolves flags with documented judgment calls. Also fits designing a missing system layer (reuse 2: Recovery), hardening a proposed architecture (reuse 3: Health Tracker), and a remote-access feature where the risk is SECURITY not tooling (reuse 4: HQ live conversation - 3 parallel lanes + a sequential adversarial security/QC 4th lane that catches what the transport lanes are naive about). |
| [verification-qa-built-artifact](verification-qa-built-artifact.md) | verification | 2 | "Is this thing I just built actually correct + complete?" Post-build QA of code + its outputs. 3 parallel lanes: static auditor (line-by-line correctness/invariants/portability) + dynamic edge-case tester (isolated fixture, inject one fault per feature) + findings verifier (re-derive outputs, hunt false negatives + self-referential bugs). Synthesize MUST/SHOULD/NICE/DEFERRED. |
| [pitch-build-adversarial-loop](pitch-build-adversarial-loop.md) | other | 1 | "Turn my system/project into a persuasion deck for a specific audience." 3 sequential loops: Loop 1 = 2 parallel research dossiers (infra + cost/value, LOGGED vs ESTIMATE), Loop 2 = QA gap-check from the audience's skeptical seat (recompute every number), Loop 3 = the Judge reads the RENDERED slides + narrative and returns a verdict + ranked design/content fixes. Session synthesizes + builds an animated branded deck between loops. |
| [venture-ideation-validate](venture-ideation-validate.md) | decision-brief | 1 | "Best N near-future business ideas out of this system/person, and are they real?" 4 sequential loops: ideate wide (10-14 candidates) -> identify (weighted score, cut to N complementary winners) -> validate (web-searched build-ready plans: verdict/offer/pricing/competitors/econ/riskiest-test/90-days/kill) -> output QC (read the rendered slides + PDF vs source before ship). Session builds briefing slides + a detailed plan PDF between loops. |
| [technical-evaluation-tool-scout-review-qc](technical-evaluation-tool-scout-review-qc.md) | technical-evaluation | 5 | "Find the best cheap tool/approach for X from the live web, and is the pick actually right?" 3 sequential agents: Scout (web/GitHub/X candidate table, LOGGED vs ESTIMATE costs) -> Reviewer (verify uncertainties, recompute cost, rank shortlist, correct overclaims) -> adversarial QC (SHIP/SHIP-WITH-FIXES, hunt the simpler path we already own). Session grounds with INTERNAL facts first so QC can catch "you already built it." Run 13 flipped adopt->evolve; run 14 variant (research-debate-review); run 16 (voice hands-free) QC deleted a framework dep + caught a TTS language gap; run 17 (upgrade scan) scaled to an 8-lane landscape sweep, QC killed a phantom headline on machine facts (settings.json + one SSH); run 22 (voice in-session) found the prior run's own kill list stale (/voice had shipped) + the QC's doc fact both earlier agents missed flipped the winner - master re-verified it before accepting. |
| [implementation-relay-plan-build-review](implementation-relay-plan-build-review.md) | other | 1 | "Implement this batch of feedback on a system we own." 3 sequential agents: Research/Processing (map every item to real code + pull real values, plan file) -> Implementation (phased execution, checkpoint-to-disk discipline, one deploy) -> Review/Validation (independent audit, fresh renders, secrets grep, BLOCKER/MINOR/NIT, signed-off checklist). Orchestrator runs the brand pre-flight and pastes tokens into the implementer's prompt. Run 15 shipped HQ v2.1 (18 items) with it. |
| [document-tailor-build-review-qc](document-tailor-build-review-qc.md) | other | 1 | "Restructure my real document (CV/profile/proposal) for a specific audience brief, branded, nothing invented." 3 sequential agents: Build (draft + render + self-verify) -> domain-expert reviewer in the audience's seat (skim test, MUST-FIX/NICE, ATS text-layer check) -> QC final judgment (independent re-verification, traceability of every number, READY or fix list). Revisions relay back to Agent 1. Key lesson: reviewers get the VERBATIM source, never a compressed excerpt (false-positive risk). Run 18 shipped the Power BI finance contract CV. |
| [deck-rebuild-content-design-qc](deck-rebuild-content-design-qc.md) | other | 1 | "Rebuild this existing deck per my page-by-page feedback, same visual language, two formats." 3 sequential agents: content pass (per-page copy doc, exact diagram labels) -> UX designer (React HTML build, static render mode, self-reviewed renders) -> QC (word-for-word diff, brand scan, pptx from fresh renders). Master gates every handoff + views all renders. Key lesson: verify the brief's source mapping against the actual files FIRST; run 20's brief pointed at the wrong deck. |
| [decision-brief-inventory-architect-debate](decision-brief-inventory-architect-debate.md) | decision-brief | 1 | "My own structure looks messy, change or keep?" 3 sequential agents: Cartographer (inventory, declared vs actual, flags only) -> Architect (exactly 3 philosophy-distinct options vs stated pain + Hcases) -> Debater (steelman/attack, eliminate to ONE + amendments, head-to-head vs the incumbent). Master reviews every hand-off vs the registry, owns the verdict. Key lesson: hand the debater the owner's REAL couplings (backup whitelist, gitignore, incident history) so eliminations land on verified facts. Run 21 (outputs/ layer) -> amended-Ledger verdict. |
| [other-system-upgrade-audit-design-plan](other-system-upgrade-audit-design-plan.md) | other | 1 | "Upgrade my whole system: review everything, design it, give me a QC'd implementation plan." Master-gated relay: Agent 1 audit as N parallel Explore lanes (flags-only, lane-prefixed IDs) -> master corrects vs ground truth -> Agent 2 upgrade design (per-project verdicts, [OWNER DECIDES] flags, rejection list) -> master standing-order sweep (binding MR2 annotations; caught a privacy leak) -> Agent 3 code-verified QC (SHIP/FIX/REJECT + missed-flag sweep) + phased plan (verification/rollback/quality bar per phase) -> master synthesis with decision batch. Recommendation-only; 4 write-first working docs. Run 24 (full Alex upgrade) -> 14-phase plan, 0 rejects. |

Rules:
- A pattern earns its file by being used, not imagined. No speculative patterns.
- Update `last_used` and `times_used` on every reuse; append to Lessons when something changed.
- If a pattern fails twice, rewrite it or delete it.
